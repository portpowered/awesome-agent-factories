## Codebase Patterns
- Root Go module is `github.com/portpowered/awesome-agent-factories` with `go 1.24`; run `go build ./internal/checks` and `go test ./...` from repository root.
- `internal/checks` is a `package main` CLI with responsibilities split across `main.go`, `readme.go`, `links.go`, `sections.go`, `alphabetical.go`, and `duplicates.go`.
- No package.json or CI tooling in this worktree; Go quality gates are `go build` and `go test ./...`.
- Use the standard Awesome badge markup: `[![Awesome](https://awesome.re/badge.svg)](https://awesome.re)` on the H1 line.
- `docs/internal/customer-ask.md` contains suggested opening copy and Contents anchor patterns for later stories.
- Resource entry format validation applies only to `- [` lines in the ten `requiredResourceSections`; use `parseResourceEntry` for downstream entry rules.
- Description rules (`description-period`, `banned-marketing-phrase`) live in `validateDescriptionRules` (links.go) with helpers in `readme.go`; banned phrases use word-boundary regex and only inspect description text, not link text.
- Duplicate detection lives in `validateDuplicates` (duplicates.go); use `collectResourceEntries` to gather parsed entries globally; `normalizeURL` lowercases scheme/host, strips fragments, and trims trailing slashes before URL comparison; names compare case-insensitively.
- Alphabetization lives in `validateAlphabeticalOrder` (alphabetical.go); use `collectSectionEntries` per resource section and compare adjacent link-text names case-insensitively; sections with zero or one parsed entry pass.
- URL hygiene lives in `validateBareURLs` and `validateTrackingURLs` (links.go); strip `[text](url)` markdown links before scanning for bare `http(s)://` URLs; tracking patterns (`utm_`, `fbclid`, `gclid`, `mc_eid`) on parsed entry URLs emit `warning` level results only.
