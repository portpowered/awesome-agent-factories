## Codebase Patterns
- Root Go module is `github.com/portpowered/awesome-agent-factories` with `go 1.24`; run `go build ./internal/checks` and `go test ./...` from repository root.
- `internal/checks` is a `package main` CLI with responsibilities split across `main.go`, `readme.go`, `links.go`, `sections.go`, `alphabetical.go`, and `duplicates.go`.
- No package.json or CI tooling in this worktree; Go quality gates are `go build` and `go test ./...`.
- Use the standard Awesome badge markup: `[![Awesome](https://awesome.re/badge.svg)](https://awesome.re)` on the H1 line.
- `docs/internal/customer-ask.md` contains suggested opening copy and Contents anchor patterns for later stories.
